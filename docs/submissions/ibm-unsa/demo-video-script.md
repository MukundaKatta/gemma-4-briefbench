# AidBriefBench Demo Video Script

Target length: 2 minutes.

## 0:00-0:15 - Problem

"Small nonprofit teams often work from messy notes, chat threads, and last-minute constraints. For this demo, I am using a volunteer food pantry routing scenario. The coordinator needs a quick plan, but also needs to see risks and assumptions before acting."

Show the live app at:

https://gemma-4-briefbench.vercel.app

## 0:15-0:35 - Load Scenario

"I click Load sample to bring in field notes about drivers, pickup sites, dropoff constraints, older phones, privacy concerns, and a Raspberry Pi available at the pantry office."

Click **Load sample**.

## 0:35-1:05 - Run Analysis

"Now I click Analyze. The app asks Gemma 4 to turn this messy context into structured JSON, not a generic chatbot answer. The UI renders that JSON into a decision brief."

Click **Analyze**.

## 1:05-1:35 - Walk Through Output

"The result has a summary, model rationale, risks, build plan, user experience notes, a DEV post angle, and assumptions. For a nonprofit workflow, the most important sections are risks and assumptions because they keep a human coordinator in control."

Scroll or point to:

- Summary
- Model rationale
- Risks
- Build plan
- Assumptions

## 1:35-1:55 - SDG Fit

"This supports SDG 2, Zero Hunger, through better food pantry logistics. It also supports small community organizations that need AI help without heavy infrastructure."

## 1:55-2:10 - Close

"The app is deployed, the source is public, and it works in demo mode for judges. With a real Gemma 4 API key or local Gemma runtime, this becomes a privacy-aware decision brief tool for small nonprofit teams."
