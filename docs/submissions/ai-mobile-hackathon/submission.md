## Project

### Idea Title

AidBrief Mobile

### Problem

Small nonprofit and volunteer teams often coordinate field operations from mobile phones, chat threads, photos, and scattered notes. A food pantry coordinator, for example, may need to plan pickups, dropoffs, driver availability, cold-storage constraints, and privacy-sensitive beneficiary details during a short time window.

The problem is not only generating a plan. The real problem is making the plan reviewable on a phone: what assumptions did the AI make, what risks still need human attention, what information is missing, and what message should be sent to volunteers?

AidBrief Mobile adapts the BriefBench workflow into a mobile-first planning assistant for community operations. A coordinator can paste or capture messy notes, run an AI analysis, and get a structured decision brief that is easy to scan on a phone.

### Key Features

- Mobile-first note input for messy operational context
- Sample nonprofit workflow for food pantry routing
- AI-generated decision brief with summary, risks, assumptions, and action plan
- Copy-ready volunteer message or coordinator brief
- Provider path for Gemini/Gemma models and future local LLM support
- Offline/local-first roadmap for privacy-sensitive field data
- Demo mode so reviewers can test immediately without API keys
